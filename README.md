# Практическая работа №1. Git и GitHub

## Цель

Научиться:

- клонировать удаленный репозиторий;
- создавать собственную ветку;
- работать с файлами;
- делать коммиты;
- отправлять изменения в GitHub;
- работать только в своей ветке.

> ⚠️ **Важно!**
>
> Работать в ветке `main` запрещено.
> Все задания выполняются **только в своей ветке**.

---

# Шаг 1. Клонирование репозитория

Склонируйте репозиторий преподавателя.

```bash
git clone git@github.com:6Katana9/KPI.git
```

---

# Шаг 2. Перейдите в папку проекта

```bash
cd kpi
```

---

# Шаг 3. Создайте собственную ветку

Название ветки должно состоять из имени и фамилии латиницей.

Примеры:

```
ivan-ivanov
```

```
sultan-talaibekov
```

Создание ветки:

```bash
git checkout -b firstname-lastname
```

---

# Шаг 4. Проверьте текущую ветку

```bash
git branch
```

Возле вашей ветки должна отображаться звездочка `*`.

# Работа с Git

После выполнения **каждого задания** необходимо сохранить изменения.

## Проверить изменения

```bash
git status
```

---

## Добавить изменения

```bash
git add .
```

---

## Создать коммит

Для каждого задания сделайте **отдельный коммит**.

Примеры сообщений:

```bash
git commit -m "Добавил информацию о себе"

git commit -m "Добавил программу Hello Git"

git commit -m "Добавил SQL запросы"

git commit -m "Добавил заметки по Git"

git commit -m "Добавил цели обучения"

git commit -m "Обновил README"
```

---

# Отправка работы

После выполнения всех заданий отправьте изменения **только в свою ветку**.

```bash
git push origin firstname-lastname
```
